# PHASE 1 — Build the Monitoring Foundation

(Highest Priority)

## Goal

Get REAL metrics into your dashboard.

---

## Step 1 — Create a Demo Service

Make a simple Spring Boot app called:

```text id="s8j4yy"
cloudpulse-demo-service
```

This simulates a production app.

Add:

* REST APIs
* fake delays
* random errors
* CPU-heavy endpoints

---

## Step 2 — Add Actuator + Prometheus

In demo service:

```xml id="mlg9nm"
spring-boot-starter-actuator
micrometer-registry-prometheus
```

Expose:

```bash id="79f18t"
/actuator/prometheus
```

---

## Step 3 — Setup Prometheus

Prometheus scrapes metrics every few seconds.

Create:

```yaml id="m79l13"
scrape_configs:
  - job_name: 'demo-service'
    metrics_path: '/actuator/prometheus'
```

Now you have REAL metrics.

---

## Step 4 — Verify Metrics

Test:

```bash id="m0l8zh"
http://localhost:8080/actuator/prometheus
```

You should see metrics like:

* JVM memory
* CPU
* HTTP requests
* response times

# PHASE 4 — Incident Engine

This makes it NOT a CRUD app.

---

## Incident Detection Rules

Example:

```text id="o9k9s8"
IF CPU > 80% FOR 2 minutes
→ Create Incident
```

Other rules:

* response time spikes
* memory leak
* high error rate
* service down

Store incidents in PostgreSQL.

---
